## Цель

Сгенерировать готовую к публикации партию LinkedIn-постов и экспертного контента,
напрямую связанную с контентными опорами, написанную на языке аудитории и переводящую
людей из осведомлённости в интерес к офферу.

## Пошаговый flow

### Шаг 1 — Спланировать партию

Перед написанием создать таблицу плана постов:

| № | Опора | Формат | Тип хука | Тема | CTA |
|---|-------|--------|----------|------|-----|
| 1 | [опора] | текст/карусель/и т.д. | любопытство/история/ценность/провокация | [тема] | [действие] |
...

Распределить посты по опорам согласно соотношению контента из стратегии.
Чередовать типы хуков — два подряд идущих поста не должны использовать один тип хука.
Включить хотя бы один пост с мягким CTA к офферу в каждой партии.

Показать план пользователю и получить подтверждение перед написанием.

### Шаг 2 — Написать LinkedIn-посты

Используя skills social-content и copywriting, написать каждый пост.

Правила для каждого поста:
- Первая строка — хук: должна создавать разрыв с паттерном. Нельзя начинать с «Я».
- Никаких маркированных списков, начинающихся с языка фич. Только пользы и результаты.
- Использовать точные фразы из `workspace/analysis/voc-quote-bank.md` минимум один раз в посте.
- Максимум 3 коротких абзаца для текстовых постов. Карусели: 5–8 слайдов.
- Каждый пост должен быть самодостаточным — без отсылок к предыдущим постам.
- Завершать либо вопросом (для стимуляции комментариев), либо мягким CTA (DM / посещение профиля).

Формат каждой записи в выходном файле:
```
---
## Пост [N] — [Название опоры]
**Формат:** [текст / карусель / история]
**Тип хука:** [любопытство / история / ценность / провокация]
**Ожидаемый охват:** [низкий / средний / высокий]

[Полный текст поста]

**CTA:** [текст CTA]
**Лучшее время публикации:** [день и время по нормам LinkedIn для B2B]
---
```

### Шаг 3 — Очеловечить все посты

Используя content-humanizer, проверить каждый пост на:
- AI-признаки: чрезмерно формальные переходы, уклончивый язык, общие советы
- Недостаток конкретности: заменить «многие профессионалы» конкретной отсылкой
- Соответствие голосу: сравнить каждый пост с brand-voice.md
- Естественный ритм: варьировать длину предложений, допускать неполные предложения там, где это органично

Переписать любой пост, который звучит обобщённо или корпоративно.

### Шаг 4 — Редактура

Используя copy-editing, провести финальную редактуру каждого поста:
- Убрать слова-паразиты (очень, действительно, просто, довольно, в некотором роде)
- Заменить пассивные конструкции
- Убрать восклицательные знаки
- Проверить: выполняет ли первая строка обещание хука
- Проверить: конкретный ли и низкофрикционный ли CTA

### Шаг 5 — Экспертный лонгрид (опционально)

Если пользователь запрашивает — сгенерировать одну экспертную статью или скрипт карусели
для наиболее приоритетной опоры.
600–900 слов для статей LinkedIn; 10 слайдов для карусели.

Применить тот же pipeline качества: social-content → copywriting → content-humanizer → copy-editing.

### Шаг 6 — Составить контент-календарь

Создать расписание публикаций для партии:

| Дата | Пост № | Опора | Формат | Время |
|------|--------|-------|--------|-------|
[Заполнить на основе размера партии и целевой частоты из стратегии]

Распределить посты равномерно по неделям. При частоте ниже 3x/неделю — не публиковать два дня подряд.

## Выходные файлы

Сохранить в `workspace/posts/`:

### linkedin-batch-[YYYY-MM-DD].md
```
# Партия LinkedIn-постов — [дата]

## Сводка партии
- Всего постов: [N]
- Охваченные опоры: [список]
- Период: [начало] — [конец]
- Тема партии: [опционально — если есть связующая нить]

[Все посты в формате, определённом в Шаге 2]
```

### content-calendar-[YYYY-MM-DD].md
```
# Контент-календарь — [дата]

[Таблица расписания публикаций из Шага 6]

## Заметки
[Временные факторы, инфоповоды, советы по вовлечённости]
```

При наличии лонгрида:

### expert-content-[YYYY-MM-DD].md
```
# Экспертный контент — [дата]

## [Заголовок]
[Полная статья или скрипт карусели]
```
